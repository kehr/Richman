# Onboarding UX Overhaul 实施计划

**目标**：按 `docs/prds/onboarding-ux-overhaul-prd.md` 的设计，给 onboarding 流程加上回退、跳过、重入、键盘导航、framer-motion 动效和浅色高级感视觉。

**执行 skill**：`superpowers:subagent-driven-development`（每个 step 派发独立 subagent，两阶段 review）。Step 文件采用 checkbox 语法（`- [ ]`）跟踪进度，但遵循项目 plan 规范不包含任何代码级细节 —— 执行 agent 根据 PRD 与现实代码动态决策实现。

## 全局规则

- 所有 step 的分支策略：在 `onboarding-ux-overhaul` 分支直接开发（当前分支即是），不开 worktree
- 每个 step 完成后必须通过 `pnpm lint:all`（前端）或 `go vet ./... && go build ./... && go test ./...`（后端），未通过不进入下一步
- 每个 step 结束产出一个 commit，commit message 遵循 conventional commits 风格
- 零 AI 痕迹原则：commit message 不含 AI/Claude 关键词，代码注释英文、不含 "generated by" 之类标记
- 执行报告放在 `docs/reports/onboarding-ux-overhaul-execution-report.md`，每完成一个 step 追加一段
- 遇到 PRD 未覆盖的决策项，先记录到执行报告，不阻塞当前 step 的实施，留给用户最后验收时决策
- 严格遵循 `docs/standards/design-review.md` 定义的纪律：修改任何文件前先读全、列既有契约、评估影响

## 验收标准

所有 step 完成后，必须满足：

- `cd frontend && pnpm lint:all && pnpm test -- --run && pnpm build` 全绿
- `cd backend && go vet ./... && go build ./... && go test ./...` 全绿
- 手动冒烟：(a) 新用户完整走完 4 步、(b) 用户任意步跳过 → dashboard 看到 nudge → 点 nudge 重入、(c) 用户通过 Settings 重新走一遍、(d) 键盘 ← / → / Esc 在 onboarding 各步的行为正确
- 执行报告 `docs/reports/onboarding-ux-overhaul-execution-report.md` 完整记录每个 step 的 commit SHA、review 结果、已修复问题、未处理观察项
- 所有新增或修改的代码通过 spec compliance review 和 code quality review 两轮 subagent 审查
