## 已记录但未修复的观察项

### 观察项 1：`janitorSweep` 存在但未 wire，存在潜在内存增长风险
- 现象：`screenshot.Service.attempts` map 会累积从未回访的 userID 条目，没有自动回收机制
- 影响：长时间运行的服务实例会有缓慢的内存增长（量级取决于独立用户数）
- 为什么本次不修：修复需要新增 background ticker 或 cron 调用 `janitorSweep`，涉及 goroutine 生命周期和测试 fixture 变化，超出「lint 清理」的 scope
- 建议：独立开一个 `fix(screenshot): wire janitor sweep to background ticker` 的 ticket

### 观察项 2：commit `0ad2ce3` 的 scope 混合已经入 history
- 现象：`onboarding-ux-overhaul` 分支的 commit message 写「docs(trd)」，实际包含 backend 修复 + 前端重构 + TRD 文档三类改动
- 为什么本次不修：用户明确选择「接受现状不重写 history」
- 缓解措施：同步起草 `docs/standards/commit-hygiene.md` 规范避免复发，并在规范的「背景」段落具名引用此次事故作为案例教学
- 后续建议：PR review 流程增加一个 checklist 项「commit message 主题是否覆盖所有暂存文件」

### 观察项 3：本机 golangci-lint v2 是 Homebrew 最新版，未固定版本
- 现象：`golangci-lint --version` -> v2.11.4，来自 `brew install golangci-lint`，但项目没有声明最低版本号
- 影响：不同开发者本地版本可能不一致，配置兼容性会再次漂移
- 为什么本次不修：涉及 CI 脚本和 Makefile 的修改，超出 scope
- 建议：按 `lint-toolchain.md` 的「工具链版本声明」段落要求，在 `backend/Makefile` 或 `tools/tools.go` 里固定 `v2.11.4`，CI 同步
