# 自动分析调度策略配置 PRD

## 背景与问题

系统当前在 `scheduler.go` 中硬编码三个分析时间窗口（A 股 08:30 / 15:30，美股 06:00），用户在持仓详情页能看到「下一次自动分析」时间，但完全无法配置何时触发、触发频率，也无法控制单个持仓是否参与自动分析。

## 目标

为用户提供三层嵌套的调度策略配置能力，细粒度层可覆盖上层设置。同时支持时间窗口自定义和高频触发选项。

## 用户故事

- 作为用户，我希望控制哪些市场时间段会触发分析，以避免在不关心的时段产生通知。
- 作为用户，我希望设置全局分析频率（如每周一次），不需要为每个持仓单独配置。
- 作为用户，我希望对个别重点持仓设置比全局更高的分析频率。
- 作为用户，我希望盘前分析包含自上次分析以来的信息增量（价格变化、相关新闻/事件）。

## 配置层级

```
全局默认 → 市场级覆盖 → 持仓级覆盖
```

细粒度层未设置时继承上层；显式设置后独立生效。

## 功能规格

### 入口 A：设置页「调度策略」Tab（全局 + 市场层）

新增第六个 Tab，Tab Key 为 `schedule`。

#### 全局默认频率

用户从以下选项中选择一个，作为所有市场和持仓的默认触发间隔：

| 选项 | 语义 |
|------|------|
| 每个窗口 | 所有已启用的时间窗口都触发，同一天可多次 |
| 每日 | 最小间隔 24h，当天只取第一个到达的窗口 |
| 每两日 | 最小间隔 48h |
| 每三日 | 最小间隔 72h |
| 每周 | 最小间隔 7 天 |
| 自定义 | 用户输入 N 天（1–30），最小间隔 N×24h |

#### 市场时间窗口

每个市场分盘前、盘后两个窗口，各自独立开关和时间自定义。

**A 股**

| 窗口 | 默认时间 | 允许范围 | 说明 |
|------|---------|---------|------|
| 盘前 | 08:30 | 07:00–09:29 | 开盘 09:30 前 1h |
| 盘后 | 15:05 | 15:00–20:00 | 收盘 15:00 后 5min |

**美股 / 黄金**

| 窗口 | 默认时间（北京） | 说明 |
|------|--------------|------|
| 盘前 | 夏令 20:30 / 冬令 21:30 | NYSE 开盘 09:30 EDT/EST 前 1h，系统自动随 DST 切换 |
| 盘后 | 夏令 04:05 / 冬令 05:05（次日） | NYSE 收盘 16:00 EDT/EST 后 5min，系统自动随 DST 切换 |

美股时间窗口随 NYSE 官方夏令/冬令时间表自动偏移，用户自定义时间以用户设置为准（不跟随 DST）。

**港股**（预留）

UI 展示占位卡片，标注「规划中」，不可交互，默认时间为盘前 08:30 / 盘后 16:05。

#### 市场级频率覆盖

每个市场可单独设置频率，默认「跟随全局」。选项同全局频率选项，额外加「跟随全局」。

#### 时间自定义交互

- 时间以按钮形式展示，点击弹出时间选择器，精度 5 分钟。
- 未修改时显示默认标记（灰色），修改后显示已修改标记（蓝色），可重置回默认。
- 关闭状态的窗口时间按钮不可点击，整行降低透明度。

### 入口 B：持仓详情「分析元信息」侧边栏（持仓层）

在现有侧边栏「分析元信息」区域增加两个配置控件：

| 控件 | 选项 |
|------|------|
| 分析频率 | 跟随市场默认 / 每个窗口 / 每日 / 每两日 / 每三日 / 每周 / 自定义 |
| 分析窗口 | 跟随市场默认 / 仅盘前 / 仅盘后 / 盘前和盘后 |

默认均为「跟随市场默认」。修改后仅作用于当前持仓。

### 盘前分析内容

盘前分析在标准分析基础上额外包含信息增量：
- 自上次分析以来的价格变动（涨跌幅、区间高低点）
- 相关新闻/事件摘要（来源同现有数据管道）

### 下次分析时间展示

「下一次自动分析」时间根据当前用户设置动态计算，优先级：持仓覆盖 > 市场设置 > 全局默认。

## 状态空间

| 状态 | 说明 |
|------|------|
| 全局频率=每个窗口，市场窗口全开 | 最高频，A 股每天 2 次，美股每天 2 次 |
| 全局频率=每日，A 股仅盘后开启 | 每天仅 15:05 触发一次 |
| 持仓频率=每周，市场频率=每日 | 该持仓每 7 天触发一次，忽略市场每日设置 |
| 全局自定义 N=5，某市场跟随全局 | 该市场每 5 天触发一次 |
| 所有窗口关闭 | 不触发自动分析 |

## 不在本期范围

- 港股数据接入（UI 预留）
- 分析结果推送频率控制（属于推送渠道配置）
- 条件触发（价格突破等事件驱动触发）

## 决策记录

- 港股 UI 预留而不实现：数据接入成本较高，优先打通配置框架，港股留到后续版本
- DST 自动感知：用户无需手动调整，系统基于 NYSE 官方 DST 时间表切换，降低认知负担
- 时间默认规则：盘前 = 开盘前 1h，盘后 = 收盘后 5min，规则化而非主观选择
- 「每个窗口」作为最高频选项而非「每小时」：投资分析不需要分钟级高频，窗口粒度与市场节奏对齐
