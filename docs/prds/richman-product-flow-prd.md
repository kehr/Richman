# Richman 产品动线设计 PRD

## 0. 文档定位

本文档是对 `richman-prd.md` 的补充，专注于**产品动线与界面设计**：用户从进入网站开始的完整路径、页面布局、交互细节。不重复 richman-prd.md 已有的产品定位、三维分析引擎数学模型、数据源选型等内容。

本文档的设计目标是把当前"6 个空菜单项"的 MVP 骨架，改造成一个有清晰动线、首次使用有 aha moment、日常回访有明确价值感的产品。

## 1. 信息架构重构

### 1.1 当前问题

现状 6 个顶级菜单项 Dashboard / Portfolio / Analysis / Decision Cards / Notifications / Settings 存在严重的职责割裂：

- Dashboard 只展示三个孤立数字（持仓数 / 总仓位 / 最近分析时间），无任何产品价值
- Analysis 是一个单按钮页，与 PRD 的"cron 自动触发"逻辑冲突
- Decision Cards 是产品核心价值，却藏在菜单第 4 项
- Notifications 实际是"推送渠道配置"，命名歧义严重
- 用户首次进入后没有任何引导，不知道从何开始

### 1.2 重构后的顶级菜单

仅保留 3 个顶级菜单项 + 1 个底部辅助入口：

| 菜单 | 路径 | 职责 |
|---|---|---|
| Dashboard | `/dashboard` | 今日决策卡墙 + 顶部瘦总览 strip + 变化锚点摘要 |
| Portfolio | `/portfolio` | 持仓列表 + 添加/编辑/交易记录/截图批量导入 |
| Settings | `/settings` | 账户 / 推送渠道 / 偏好 / 订阅 四个内部 tab |
| 帮助（底部） | `/help` | 术语、徽章状态、维度、建议动作、信心度、数据源说明 |

被取消的菜单项及其去向：

- `Analysis`：取消。三维分析由后台 cron 触发（08:30 / 15:30 / 次日 06:00 CST），用户需要主动刷新时点击 Dashboard 顶部"重新分析"按钮对所有持仓重跑
- `Decision Cards`：取消。Dashboard 本身就是决策卡墙；单卡详情通过点卡进入 `/decision-cards/:id`，不占菜单位
- `Notifications`：取消。渠道配置合并到 `Settings → 推送渠道` tab

### 1.3 完整 sitemap

```
公开区（未登录）
  /login                            登录页 + 左侧产品介绍 + 样例决策卡截图
  /register                         邀请码注册

首次登录强制 onboarding
  /onboarding/welcome               第 1 步 欢迎与理念
  /onboarding/categories            第 2 步 选关注的标的类型
  /onboarding/first-holding         第 3 步 录入第一个持仓（快速 / 明细 / 截图）
  /onboarding/first-analysis        第 4 步 触发首次分析进度屏

主应用区
  /dashboard                        首屏决策卡墙
  /decision-cards/:id               决策卡详情页（点卡 / 推送回流目标）
  /portfolio                        持仓列表
  /portfolio/:id/transactions       交易记录子页（明细模式买卖流水）
  /settings                         设置页（内部 tab 切换）

```

## 2. 入口与首次 Onboarding

### 2.1 登录页 `/login`

- 左右双栏布局（桌面 1200+ 宽度）
  - 左侧（占 60%）：产品名 Richman + 定位标语"把基金经理的思维方式装进你的口袋" + 三行简介 + 一张样例决策卡截图（示意三维 badge + 建议 + 信心度）
  - 右侧（占 40%）：登录表单（邮箱 + 密码 + 登录按钮 + "用邀请码注册"小字链接）
- 移动端折叠为单列，介绍在上、表单在下
- 登录页兼作"极简落地页"：不做完整营销页，但给意外访问的人足够线索理解产品
- URL 支持 `?returnTo=<path>`，登录成功后跳回原路径，用于推送链接未登录回流

### 2.2 注册页 `/register`

- 邮箱 + 密码 + 邀请码 三字段
- 邀请码校验失败时提示"邀请码无效或已用完"
- 注册成功后直接进入 `/onboarding/welcome`

### 2.3 Onboarding 4 步强制流程

注册后或首次登录时，未完成 onboarding 的用户被强制跳入 `/onboarding/welcome`。完成标记写入用户记录后永不再进入。

**第 1 步 `/onboarding/welcome` — 欢迎与理念**
- 单列居中布局
- 标题"欢迎，让我们开始"
- 一段 3 行的产品理念陈述（基于持仓 / 三维分析 / 明确建议）
- 三个小卡片平铺简介三维（趋势 / 位置 / 催化剂）
- 底部大按钮"开始设置 →"

**第 2 步 `/onboarding/categories` — 选关注的标的类型**
- 多选 2x2 网格卡片：黄金 ETF / A 股宽基 ETF / 行业 ETF / 美股
- 每张卡片显示类型名 + 一行常见标的示例
- 选中状态为 2px 黑边 + 浅灰底
- 至少选 1 个才能继续
- 选择结果用于下一步标的库筛选

**第 3 步 `/onboarding/first-holding` — 录入第一个持仓**
- 三个 tab 切换：快速模式 / 明细模式 / 截图识别
- 快速模式（默认）：标的选择 + 均价成本 + 仓位比例 3 个字段
- 明细模式：标的选择 + 增减交易记录表格（可多条）
- 截图识别：上传按钮 → 识别结果展示（见 4.4 截图校对）
- 底部小字软链接"先录 1 个就行，后面随时可以加"，没有"跳过"按钮
- 录完后底部大按钮"保存并开始分析 →"
- 底部可选小字链接"想看具体金额？设置总资金（可选）" 打开总资金填写浮层

**第 4 步 `/onboarding/first-analysis` — 触发首次分析进度屏**
- 单列居中
- 显示 4 步分析进度（拉行情 → 计算量化底座 → LLM 催化剂搜索 → 生成决策卡）
- 每步完成显示 ✓ 绿色，进行中显示动效，未开始灰化
- 完成后自动跳转 `/dashboard` 并展示首张决策卡

## 3. Dashboard 核心首屏

`/dashboard` 是用户每次回来的默认落地页，也是 Onboarding 完成后的第一个"主应用页面"。

### 3.1 三区结构

**顶部瘦 strip（白底）**
- 第一行：
  - 左：大标题"今日决策" + 副标题"最后分析 HH:MM · 下次自动 HH:MM"
  - 右：黑色主按钮"⟳ 重新分析"（对所有持仓重跑）
- 第二行（用细线分隔）4 列总览指标：
  - 持仓数（3 / 5）
  - 总资金（未设置时隐藏本列或显示"设置以查看"）
  - 综合浮盈亏（百分比 + 金额，如 `+4.82% · ¥ +4,820`）
  - 已分配仓位（百分比 + 金额，如 `70% · ¥ 70,000`）
- 风险声明不在此处重复，放在决策卡详情页的 meta 栏

**中部决策卡墙（浅灰底）**
- 响应式网格：3 列（≥1200px）/ 2 列（900-1199px）/ 1 列（<900px）
- 每张卡对应一个持仓，点击进入 `/decision-cards/:id`
- 空状态：持仓为 0 时整页变成一张大号引导卡"先添加一个持仓"，顶部 strip 和底部变化区都不渲染

**底部"变化锚点摘要"（白底）**
- 只列出本次分析有变化的卡（徽章不为空的卡），每条一行
- 格式：`● [徽章类型] [标的名] → [变化摘要]`
- 点击某条 → 滚动到 Dashboard 中对应的决策卡并高亮 1.5 秒
- 所有卡都无变化时隐藏整个区域

### 3.2 决策卡组件（Dashboard 摘要版）

每张卡从上到下 6 个区域：

1. **头部**：左侧类型标签 + 标的名 + 代码 · 成本 · 仓位 [· ¥ 金额]；右侧大字浮盈亏百分比 + 现价 [+ ¥ 变动金额]
2. **三维 badge**：趋势 / 位置 / 催化剂 三个 chip，翻转的维度用彩色底（绿 / 红）+ 方向箭头 + 划掉的旧值
3. **建议框**（灰底，有变化时左侧 3px 色边）：
   - 顶部：`建议` 标签 + 右侧`目标仓位 XX% [· ¥ X]`
   - 中部：建议文案 `小幅加仓 · 分 3 批`（有变化时附 `← 旧建议` 小字）
   - 底部虚线以下：执行计划摘要条（见 3.3）
4. **今日要点**：1-2 行灰色文字，说明"为什么变了" / "为什么这个建议"
5. **底部分隔**：左侧 `信心度 XX% [(+N) / (-N)]`；右侧 `查看完整推理 →`
6. **右上角变化徽章**（见 3.4）

### 3.3 执行计划摘要条（卡面版）

位于建议框下方，虚线分隔：

- 默认展示前 3 步
- 每步一行，格式 `[编号圆圈] 触发条件 [加粗] · 变动仓位 [加粗，附金额]`
  - 编号圆圈颜色递减：实心黑 / 深灰 / 中灰
  - 触发条件样式：价格（`≤ 4.05`）/ 时点（`财报后 2 周`）/ 事件（`Fed 议息后`）
- 持有场景无编号步骤，显示两条监控线 `· 跌破 X → 检视` `· 涨破 Y → 止盈`
- 一次性场景保持编号 `1`，仅一行
- 超过 3 步时，第 3 行下方显示 `+ 还有 N 批，查看详情 →` 链接

### 3.4 变化徽章状态机

每张决策卡右上角显示一个变化徽章。后端基于本次与上次分析的 diff 计算徽章类型。用户扫一眼徽章颜色就能判断"这张卡发生了什么类型的事情"。

**8 种状态定义**：

| 状态 | 颜色 | 条件 |
|---|---|---|
| 数据降级 | 灰 `#6B7280` | 至少一个数据源降级，`analysis.degraded = true` |
| 首次分析 | 黑 `#000` | 此持仓无上一次分析可对比 |
| 建议升级 | 绿 `#10B981` | `action` 积极度等级提升（如持有→加仓） |
| 建议降级 | 红 `#EF4444` | `action` 积极度等级下降（如加仓→持有） |
| 信号翻转 | 蓝 `#3B82F6` | 至少一个维度方向翻转，但 `action` 未变 |
| 计划调整 | 琥珀 `#F59E0B` | `action` 未变、维度未翻转，但 `execution.steps` 或 `target_position_pct` 有变化 |
| 信心度波动 | 紫 `#8B5CF6` | 其他都没变，`abs(confidence_delta) ≥ 10` |
| 无徽章 | — | 完全无可感知变化 |

**优先级**（同时命中多个时取最高）：
数据降级 → 首次分析 → 建议升级/降级 → 信号翻转 → 计划调整 → 信心度波动

**建议积极度等级**（用于升降级判断）：
```
2  积极加仓
1  小幅加仓
0  持有等待
-1 分批减仓
-2 控制仓位
```
升级 = 新等级 > 旧等级，降级 = 新等级 < 旧等级。

### 3.5 建议数据模型

后端 API 返回的决策卡 `recommendation` 字段改为结构化对象，前端据此渲染建议框和执行计划：

```typescript
interface Recommendation {
  action: 'add_big' | 'add_small' | 'hold' | 'reduce' | 'clear';
  action_level: 2 | 1 | 0 | -1 | -2;              // 用于升降级判断
  label: string;                                    // 文案，如 "小幅加仓"
  target_position_pct: number;                      // 目标仓位
  current_position_pct: number;                     // 当前仓位
  execution: {
    type: 'one-shot' | 'staged' | 'monitor';
    steps?: Array<{
      order: number;
      trigger_type: 'price' | 'time' | 'event';
      trigger_value: string;                         // 文案表达，如 "≤ 4.05"、"财报后 2 周"、"Fed 议息后"
      trigger_payload?: {                            // 结构化触发条件（用于前端渲染 / 未来自动提醒）
        price_op?: 'lte' | 'gte';
        price_value?: number;
        deadline_iso?: string;
        event_key?: string;
      };
      delta_pct: number;                             // 本批变动仓位
      rationale: string;                             // 详情页展示的完整推理
    }>;
    stop_loss?: number;                              // 持有场景的止损价
    take_profit?: number;                            // 持有场景的止盈价
  };
}
```

**约束**：
- `steps.length` 无上限，由后端根据分析动态决定
- 卡面显示前 3 步，超过 3 步在卡面显示折叠链接
- `type = 'monitor'` 时不使用 `steps`，而是 `stop_loss` / `take_profit`

## 4. Portfolio 持仓管理

### 4.1 Portfolio 列表页 `/portfolio`

- 顶部标题"我的持仓" + 副标题"3 / 5 个 · MVP 每用户最多 5 个标的"
- 顶部右侧两个按钮：
  - 次按钮"📷 截图批量导入"
  - 主按钮"+ 添加持仓"（达到 5 上限时置灰 + hover 提示）
- 顶部第二行（若用户已设置总资金）：`总资金 ¥ 100,000 · 修改`，未设置时显示 `未设置总资金 · 设置以查看金额`
- ProTable 风格持仓表格，列：
  - 标的（名 + 代码）
  - 类型
  - 成本
  - 现价
  - 仓位（百分比，已设总资金时附金额）
  - 浮盈亏（百分比 + 金额）
  - 操作（编辑 / 交易记录 / 删除）
- 点击行（非操作列）= 进入该标的最新一张决策卡详情

### 4.2 添加持仓 Drawer

点击"+ 添加持仓"打开从右侧滑入的 Drawer，宽度 520-720px。

**两步流程**：

**Step 1 选择标的**
- 类型 tab 栏：黄金 / A 股宽基 / 行业 / 美股
- 搜索框：按名或代码搜索
- 下方标的卡片网格：每张显示名 + 代码 + 一行简介
- 选中后自动进入 Step 2

**Step 2 填写信息**
- 顶部"已选标的"chip：显示名 + 代码 + 类型 + "更换"链接
- 三个 tab 切换：
  - **快速模式**：均价成本 + 当前仓位比例 2 个字段 + 底部灰底提示文案
  - **明细模式**：可增减行的交易记录表格（时间 / 价格 / 数量 / 方向 4 列），底部自动计算"综合成本 X · 综合仓位 X%"
  - **截图识别**：上传框 + 识别中状态 + 识别结果预填快速模式表单（可手动修正）
- 三个 tab 的数据共享同一个 holding 对象，切换 tab 时保留已填内容
- 底部右侧"取消"+ 黑主按钮"保存"

**进度指示**：顶部显示 `✓ 选择标的 — ② 填写信息`，支持点击回退到 Step 1

### 4.3 截图批量导入全屏 Modal

点击"📷 截图批量导入"打开全屏 Modal。

**布局**：
- 顶部：暗色（`#1F1F1F`）header，左侧标题"截图识别结果 — 校对" + 副标题"识别出 N 个标的 · 请检查高亮字段后确认导入"；右侧关闭按钮
- 主区域左右双栏：
  - 左栏（38% 宽）：用户上传的原始截图，可点击放大到全屏查看
  - 右栏（62% 宽）：可编辑的识别结果表格，每行一个标的
- 表格列：标的（名 + 代码输入框）/ 成本 / 仓位% / 删除按钮
- **低置信字段**黄色底色高亮（`#FEF3C7`），输入框边框橙色 `#F59E0B`
- 表格下方黄色提示条 `⚠ 标黄行的字段识别置信度较低，请核对`
- 底部左侧说明"将覆盖当前 N 个持仓（仓位总和 = X%）"或"将新增 N 个持仓"
- 底部右侧"取消"+ 黑主按钮"确认导入 N 个"

**行为**：
- 识别由后端 LLM Vision 接口完成，字段 `confidence` 控制高亮
- 确认导入前允许用户任意修改所有字段，包括增删行
- 导入后跳回 Portfolio 列表，顶部 toast 提示"成功导入 N 个持仓"

### 4.4 交易记录子页 `/portfolio/:id/transactions`

- 顶部面包屑 `← Portfolio / 沪深 300 ETF / 交易记录`
- 表格列：时间 / 价格 / 数量 / 方向（买入 / 卖出）/ 操作（编辑 / 删除）
- 右上"+ 添加交易记录"按钮
- 底部汇总条：综合成本 X · 综合仓位 X% · 总买入金额 X · 总卖出金额 X
- 只有通过明细模式录入的持仓才会在这里显示数据

### 4.5 5 持仓上限

- 达到 5 上限时"+ 添加持仓"主按钮置灰
- Hover 提示"MVP 每用户最多 5 个标的，请先删除或替换"
- 截图批量导入识别出超过剩余额度时，在确认前警告"你当前有 3 个持仓，截图包含 5 个新标的，超出 5 个上限，请取消选择部分行或先删除现有持仓"

## 5. 决策卡详情页 `/decision-cards/:id`

决策卡详情页是 Dashboard 卡点击 / 推送链接回流 / 此标的历史分析链接的共同目标页面。它承载单张卡的完整推理。

### 5.1 布局

- 顶部面包屑 `← Dashboard / 决策卡 / 沪深 300 ETF · YYYY-MM-DD HH:MM`
- 主内容区 + 右侧固定 meta 栏（260px 宽，1024px 以下折叠到顶部）

### 5.2 主内容区 5 个区块

**区块 1 头部 Hero**
- 左：类型标签 + 代码 + 标的大标题（28px）+ 一行 `成本 X · 现价 Y · 仓位 Z [· ¥ 金额]` + 浮盈亏着色
- 右：变化徽章（与 Dashboard 卡右上角同款）

**区块 2 结论横幅**
- 左：`今日建议`标签 + 大字建议 `小幅加仓 · 分 3 批` + 叙述行 `从当前 30% 逐步加到目标 38% · 较上次"持有等待"升级`
- 右：`信心度`标签 + 大字百分比（28px）+ delta 小字 `+7 vs 上次`
- 左侧 4px 色边（绿 / 红 / 黑，根据徽章状态）

**区块 3 执行计划**
- 标题"执行计划" + 右侧图例 `触发方式 · 本批变动 · 推理依据`
- 每一批独立卡片，从上到下：
  - 头部：编号圆圈（颜色按 order 递减） + 触发条件（加粗）+ 右侧变动仓位
  - 推理区（`为什么选这个触发值` / `为什么是这个比例` / `前置条件` / `触发不到会怎样` / `时间窗口`）— 这些字段来自 `step.rationale` 解析
- 最后显示黄色提示条：本计划的有效期（例 `本计划适用未来 7 天，超过或触发止损条件会自动重新分析`）
- 持有场景：不显示步骤卡片，替换为两条监控线（`止损 / 止盈`）各一行
- 一次性场景：只显示 1 张步骤卡片

**区块 4 三维推理**
- 标题"三维推理"
- 每个维度独立卡片，从上到下：
  - 头部：维度名 + 状态 chip（与卡面同款，含翻转箭头）+ 右侧权重微调轨迹 `权重 X% → Y%`
  - 量化信号详情（趋势：MA / RSI / MACD / 动量评分；位置：PE 分位 / PB 分位 / 北向资金；催化剂：Polymarket / 关键词匹配 / LLM 搜索摘要）
  - 文字结论
- 翻转的维度加彩色边框（绿 / 红）+ 浅色底，底部加 📌 注释 `此维度发生翻转是本次建议升级的主要驱动因素`

**区块 5 主要风险**
- 黄色告警底（`#FEF3C7` + `#92400E` 文字）
- 1-2 条主要风险，每条加粗的风险类型 + 一段详细说明
- 底部加一行"终止本计划的条件"

### 5.3 右侧 meta 栏

- 分析时间（含时区）
- 数据源状态逐条（AKShare / Polymarket / LLM 增强），降级时红色 + 说明
- 下次自动分析时间
- 此标的历史分析列表（最近 3-5 次简短列表：时间 / 建议 / 信心度），底部"查看全部 →"
- 风险声明（统一一次，不在其他位置重复）

### 5.4 未登录回流

- 推送链接 → 落地到 `/decision-cards/:id` 详情页
- 未登录时重定向到 `/login?returnTo=/decision-cards/:id`
- 登录成功后自动跳回原路径

## 6. Settings 设置页 `/settings`

### 6.1 布局

- 顶部页面标题"设置"
- 左侧 tab 栏（200px 宽）+ 右侧内容区
- 当前 tab 左侧 3px 黑边 + 浅灰底
- 4 个 tab：账户 / 推送渠道 / 偏好 / 订阅与额度

### 6.2 Tab 1 · 账户

- 邮箱（只读）
- 修改密码：按钮"发送修改链接到邮箱"（不在页内改密，保持安全模型简单）
- **总资金**（可选）：数字输入框 + 货币标签 "CNY" + 保存按钮 + 隐私提示小字
- **风险偏好**：下拉选择（稳健 / 中性 / 激进），带说明小字 `影响 LLM 在权重微调范围内的倾向`
- 退出登录按钮（红色次按钮）

### 6.3 Tab 2 · 推送渠道

原独立 Notifications 菜单项迁移至此。

- 顶部渠道数量 `当前已启用 N 个渠道`
- 已启用渠道列表：每行显示类型图标 + 配置摘要（如 "邮件 · kyle@example.com"）+ 启用开关 + 测试发送按钮 + 删除按钮
- "+ 添加渠道"按钮打开 Drawer：
  - Step 1 选择渠道类型（微信公众号 / 飞书机器人 / 邮件，后续扩展钉钉 / Slack / Telegram）
  - Step 2 填配置表单（字段随渠道不同）
- 底部说明文案：`推送时段：北京时间 08:30 / 15:30 / 次日 06:00，根据持仓自动筛选` 附带 `/help#push` 链接

### 6.4 Tab 3 · 偏好

- 语言：单选（中文 / English）
- 时区：下拉（默认 Asia/Shanghai，可切换）
- 主题：固定"亮色"，显示 `MVP 暂不支持暗色` 灰字
- 数字格式（可折叠的高级选项）：千分位分隔符 / 货币符号位置 / 小数位数

### 6.5 Tab 4 · 订阅与额度

- 当前订阅：显示"invite"徽章 + "邀请用户"说明
- 额度使用网格：
  - 持仓数（3 / 5）
  - 每日分析次数（invite 无限制）
  - 可用推送渠道（2 / 3）
  - LLM 模型（Claude Sonnet 4.6）
- "申请升级"按钮置灰显示"敬请期待"

## 7. 帮助页 `/help`

### 7.1 入口

- 左侧菜单底部一个独立的"? 帮助"入口（不占顶级菜单位）
- 决策卡和详情页的各种状态标签、维度名、信心度旁的 `?` 小图标：hover 显示 tooltip，点击跳到 `/help#anchor` 对应锚点
- Settings 的推送时段说明、风险偏好说明等小字链接也指向 `/help` 的对应锚点

### 7.2 内容结构

单页长文档 + 左侧锚点导航，章节：

1. **变化徽章** `#badge`：8 种状态定义 + 优先级规则 + 示例截图
2. **三维分析** `#dimensions`：趋势 / 位置 / 催化剂 的详细说明，每个维度列出量化信号和输出含义
3. **建议动作** `#actions`：5 个动作（积极加仓 / 小幅加仓 / 持有等待 / 分批减仓 / 控制仓位）的定义和对应积极度等级
4. **执行计划** `#plan`：staged / one-shot / monitor 三种类型的说明，触发方式（价格 / 时点 / 事件）的定义
5. **信心度** `#confidence`：计算逻辑、降级规则、波动阈值
6. **数据源** `#data`：AKShare / Yahoo Finance / Polymarket / LLM 增强的职责和更新频率
7. **降级策略** `#degradation`：哪些情况会触发数据降级，对分析结果的影响
8. **推送时段** `#push`：三个时段的覆盖规则、时区、跳过规则
9. **风险偏好** `#risk`：稳健 / 中性 / 激进 对权重微调的影响

## 8. 总资金功能（隐私敏感字段）

### 8.1 功能定义

用户可选设置一个总资金数字，设置后所有百分比位置都自动附带 ¥ 金额显示，避免用户心算。未设置时所有位置只显示百分比。

### 8.2 数据模型

```sql
-- users 表新增字段
total_capital_cny NUMERIC(18, 2) NULL,
```

MVP 阶段仅支持 CNY，不做币种区分。未来多币种扩展时扩展为 JSONB 或独立表。

### 8.3 入口

1. Portfolio 页顶部主入口（未设置时显示"设置以查看金额"，已设置显示金额 + "修改"）
2. Onboarding 第 3 步底部可选小字软入口
3. Settings → 账户 tab 的字段

### 8.4 显示位置

设置后以下位置自动附加 ¥ 金额：

- Dashboard 顶部 strip：新增"总资金"列 + 综合浮盈亏金额 + 已分配仓位金额
- Dashboard 决策卡：浮盈亏金额 + 仓位金额 + 目标仓位金额 + 执行计划每批金额
- 决策卡详情页：同上
- Portfolio 列表：仓位列附金额 + 浮盈亏列附金额

### 8.5 隐私保护（强制要求）

- 总资金**不进入** LLM 分析上下文
- 总资金**不入** 分析日志
- 总资金**不出现在** 推送渠道的消息内容中（微信模板消息、飞书卡片、邮件）
- 分析引擎只读百分比，保证"设了总资金"和"未设"的用户得到完全一致的分析结果
- 金额换算在前端渲染时完成，或后端 API 响应层附加 `amount` 字段（具体实现在 TRD 决定）
- Portfolio 顶部和 Settings 账户 tab 都显著展示一行说明 `总资金仅本地保存用于金额换算，不会进入任何分析请求`

## 10. 非目标与边界

以下内容明确**不在**本次动线设计范围：

- AI 对话追问（决策卡详情页不提供 chat 入口，MVP 纯展示）
- 社交分享（不生成分享链接、不做 open graph）
- 历史决策胜率回测（不统计"过去的建议准不准"）
- 多币种总资金（MVP 仅 CNY）
- 持仓导入（Excel / CSV / 券商 API）
- 移动端 Native App（Web 响应式足够）
- 支付集成与订阅升级（Settings 订阅 tab 只展示不操作）
